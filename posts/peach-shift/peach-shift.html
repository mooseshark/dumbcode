Convert a String to Another String Using the Bitwise XOR Operation

So, curiousity has gotten the best of you? You want to know what a bitwise exclusive or opertator is? Maybe you already know and are thinking, "This isn't how you use that"? Perhaps you just want to see how dumb this can be. Please, allow me to take you on a magical jounrey of pointlessness.

What even is this?

This is a demonstration of how to use the bitwise exclusive or opertator to convert one string to another and the process that goes into doing so.

Why would you ever do that?

Really, you wouldn't. There's basically no scenario where this would ever be useful or convenient. But, that doesn't mean it can't be interesting. This is a neat way to see how the bitwise exclusive or (XOR) operator works. This will allow us to take 2 strings, break them down to decimal values, break them down again to binary values, then use a few extra steps to rebuild our initial string to the string we wish to convert it to.

I don't know? That sounds pretty stupid.

It is pretty stupid, but there's some smart XOR operator stuff shuffled around in all the stupid. So, first, What's a bitwise exclusive or operator? It's purpose is to take two binary values and compares them. The return of this comparison results in a 1 anywhere the two binary values do not match, and a 0 where they do. The XOR operator is signified by a caret (^) in the majority of programming languages, if not all of them.

Alright, how about an example?

Alright, let's break this down. We'll start with 2 characters p and s. p is represented by the binary value 1110000. s is represented by the binary value 1110011. So when we use the bitwise XOR operator, we are writing it as p ^ s, however, the computer will see this is 1110000 ^ 1110011. The computer will then compare these 2 binary numbers to see which bits are the same, and which ones are different. In this case, 11100 are all the same. This will produce five 0 bits. The last two bits of p and s are differnt. This will produce two 1 bits. That makes the final return of 1110000 ^ 1110011 equals 0000011 or simply 11. Unforetunately, 11 is difficult to represent as an ASCII character as it stands for "End of Text". ASCII conversion aside, this still makes for a decent example as to how bitwise XOR functions.

Tight! Let's waste some time converting strings with XOR. We've got some of the building blocks in the table above, but it doesn't really paint the entire picture. For the purpose of this, we're going to take two strings I've chosen: peach and shift. Why these two? I like Princess Peach, we're shifting some bits around, they both have 5 characters. As stated before, the XOR operator does a comparison of two binary values and returns true (1) or false (0) depending on if the bits in those two values are different or the same. We have to do a little bit of hoop jumping to take a string, break it down to binary values for each individual character, then rebuild those binary values as our intended result. We'll start with our strings, peach and shift. Both strings have to be broken down to individual characters. The way I've done this is using a for in loop and a counter. But there's lots of other, likely better ways to do this. You could use the string .split() function or a normal for loop to go over each character. It's really up to you as long as we have access to the individual characters of our two strings. 


  let stringOne = "peach";
  let stringTwo = "shift";
  let counter = 0;
  let stringTwoLen = stringTwo.length - 1;

  for (let s in stringOne){

    //counter is reset to 0 when the length of stringOne
    //exceeds the length of stringTwo, forcing the
    //conversion to start over from the beginning for the
    //remaining characters in stringOne
    counter === stringTwoLen ? counter = 0 : counter++;
  }

Now that we have both strings seperated out we can start our bitwise XOR operation the characters right? That's a big negative. Well, at least in JavaScript that's a big negative. To perform a bitwise XOR operation on a string, we need to conver the string character to a decimal value. A decimal value is a numerical representation for an ASCII character. In our case, p will become 112, and s will become 115. We can find the decimal value of a character in JavaScript using the .charCodeAt(index) function, where index is the location of our character in the string. Since we've broken down our strings into single characters, our index will always be 0.

Cool. We've reached our first XOR operation. Similar to the above example, we'll now take the decimal values, do a bitwise XOR comparison on them and receive a result. Sticking with p and s, we ended up with an operation that looks like 112 ^ 115. And like before, this breaks down further to 1110000 ^ 1110011, leaving us with a final result of 11.

We did it! Mission accomplished.

We've just gotten started with how overly complicated this gets. Now we need to take the first character of our initial string and do a secondary bitwise XOR operation with the result of our first bitwise XOR operation. That's right, we're doubling down! We start with the decimal value of p from peachwhich is 112. Next We take our result from our first XOR operation 11, execpt 11 is the binary value the computer sees from comparing the bits between the two decimal values. What we will see is the decimal value rpresented by 11, which is 3. As stated before, 11 or in this case 3 is the ASCII representation for "End of Text" which isn't really easy to show as an ASCII character. Now that we have these two decimal values, we do our bitwise XOR operation 112 ^ 3. Naturally, the computer breaks this down to a binary operation of 1110000 ^ 0000011 which results in 1110011. And you know what that converts to? 


    let stringOneDecimal = stringOne[s].charCodeAt(0); //112
    let stringTwoDecimal = stringTwo[counter].charCodeAt(0); //115
    let firstXORDecimal = stringOneDecimal ^ stringTwoDecimal; //3

    //Now we redo the operation to get back to the decimal
    //value from stringTwo, or the string we wish to convert to
    let secondXORDecimal = stringOneDecimal ^ firstXORDecimal; //115

THAT'S RIGHT! We're back at decimal value 115. In JavaScript, we can use String.fromCharCode(115) back to the ASCII character represented by 115 and of course we get s. And through a needlessly complicated and incredibly stupid method, we've converted one character from one string to another character of another string. Now that the character has been converted we can append it to a new strong, or the way I did it, push the character to an array. We then repeate the process for each character in each string. The end result is either a full strong, or an array container the characters of the converted string. With the array method, we can then use ourArray.join("") to recombine all of the characters back to a single string. Our end result being the string shift.

Wow... I've wasted my life.

Not so fast now. I did a little bit of research, just to make sure I wasn't missing something here. It turns out this exact method can be used as some sort of low level encryption. That's something I 100% didn't know when I wrote it. I thought I just found a really useless complicated thing I could do with a bitwise operator. Essentially you take two strings, your message to encrypt, and a key word for unencrypting the message. Just as we've done here, the message is encrypted by taking the key and using the bitwise XOR operator to change the original message into a garbled mess. When you give the encrypted message to someone, they use the key given to them to decypt the message. Basically converting a string to another string. Pretty neat really.

So that wasn't really a complete waste of my life?

I guess not, but honestly, this method of encryption is pretty weak. This article basically covered everything needed to uncover the message. If you could decrypt a few words or a sentance of the message, you'd probably be able to start taking a guess at what the decryption key was. It appears this method can be repeated though, and the more iterations of encrypting a message, the harder it will be to decrypt.

    function peachShift(originalString, covertString){
    	let stringOne = originalString.toString();
    	let stringTwo = covertString.toString();
    	let counter = 0;
    	let stringTwoLen = stringTwo.length - 1;
    	let finalString = [];

    	for (let s in stringOne){
    		let stringOneDecimal = stringOne[s].charCodeAt(0);
                let stringTwoDecimal = stringTwo[counter].charCodeAt(0);
    		let firstXORDecimal = stringOneDecimal ^ stringTwoDecimal;
    		let secondXORDecimal = stringOneDecimal ^ firstXORDecimal;
    		let convertedCharacter = String.fromCharCode(secondXORDecimal);

    		finalString.push(convertedCharacter);

    		counter === stringTwoLen ? counter = 0 : counter++;
    	}

    	return finalString.join("");
    }

    function peachEncrypt(){
      let message = document.getElementById('originalMessage').value;
    	let key = document.getElementById('encryptionKey').value;
      let messageString = message.toString();
      let keyString = key.toString();
      let counter = 0;
      let keyLen = keyString.length - 1;
      let finalString = [];

      for (let s in messageString){
        let messageStringDecimal = messageString[s].charCodeAt(0);
        let shifted = messageStringDecimal * 10;
        let keyStringDecimal = keyString[counter].charCodeAt(0);
        let firstXORDecimal = shifted ^ keyStringDecimal;
        let convertedCharacter = String.fromCharCode(firstXORDecimal);

        finalString.push(convertedCharacter);

        counter === keyLen ? counter = 0 : counter++;
      }

      return finalString.join("");
    }
