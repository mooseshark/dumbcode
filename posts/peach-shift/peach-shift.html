Convert a String to Another String Using the Bitwise XOR Operation

So, curiousity has gotten the best of you? You want to know what a bitwise exclusive or opertator is? Maybe you already know and are thinking, "This isn't how you use that"? Perhaps you just want to see how dumb this can be. Please, allow me to take you on a magical jounrey or pointlessness.

What even is this?

This is a demonstration of how to use the bitwise exclusive or opertator to convert one string to another and the process that goes into doing so.

Why would you ever do that?

Really, you wouldn't. There's basically no scenario where this would ever be useful or convenient. But, that doesn't mean it can't be interesting. This is a neat way to see how the bitwise exclusive or (XOR) operator works. This will allow us to take 2 strings, break them down to decimal values, break them down again to binary values, then use a few extra steps to rebuild our initial string to the string we wish to convert it to.

I don't know? That sounds pretty stupid.

So, now that we've clarified that this is pretty dumb, let's move on to talking about the smart stuff used to do something so dumb, the XOR operator. The XOR operator is signified by a caret (^) in the majority of programming languages, if not all of them. It's purpose is to take two binary values and compares them. The return of this comparison result in a 1 anywhere the two binary values do not match, and a 0 where they do.

How about an example?

Alright, let's break this down. We'll start with 2 characters p and s. p is represented by the binary value 1110000. s is represented by the binary value 1110011. So when we use the bitwise XOR operator, we are writing it as p ^ s, however, the computer will see this is 1110000 ^ 1110011. The computer will then compare these 2 binary numbers to see which bits are the same, and which ones are different. In this case, 11100 are all the same. This will produce five 0 bits. The last two bits of p and s are differnt. This will produce two 1 bits. That makes the final return of 1110000 ^ 1110011 equal 0000011 or simply 11. Unforetunately, 11 is difficult to represent as an ASCII character as it stands for "End of Text". ASCII conversion aside, this still makes for a decent example as to how bitwise XOR functions.
